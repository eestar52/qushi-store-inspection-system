# 衢食严选 · 门店巡检管理系统

> 本项目为产品经理求职作品集，记录了一款面向衢州本地连锁餐饮品牌的 B 端 SAAS 产品从 0 到 1 的完整设计过程，涵盖需求分析、竞品研究、原型设计、数据分析、PRD 文档等核心交付物。

---

## 项目背景

**衢食严选**成立于 2018 年，总部位于浙江省衢州市柯城区，是衢州本地连锁快餐品牌，旗下门店 57 家，覆盖柯城区、衢江区、龙游县、江山市、常山县。

在数字化改造前，门店巡检依赖纸质表单，存在以下核心痛点：

- 区域经理每周人工汇总数据耗时约 **3 小时**，数据严重滞后
- 整改追踪全靠企业微信群，整改完成率不足 **50%**
- 每月门店巡检覆盖率仅 **30%**，食品安全隐患较高
- 无历史数据沉淀，无法支撑精细化运营决策

## 产品核心亮点

### 1. 私有化部署，数据不出境

餐饮行业对经营数据高度敏感，衢食严选选择本地服务器 + 阿里云备份的私有化方案，这也是相较于外部 SaaS 产品（如云巡检）的核心差异化。

### 2. 数据驱动迭代

V1.1 的两个核心功能（智能预警、截止提醒）均由数据分析结论直接驱动——厨房卫生问题占比 42% 且完成率仅 43%，店长「不知道截止日期」是整改完成率低的主因，这些发现直接转化为功能需求。

### 3. 整改闭环设计

整改工单要求上传整改后照片（系统强制校验），配合截止提醒推送和验收流程，形成「发现 → 工单 → 整改 → 验收」的完整闭环，而不只是「发现 → 通知」。
